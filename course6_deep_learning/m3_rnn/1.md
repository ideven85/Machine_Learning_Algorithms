> > > as speech, text, or time series data.

**Key Characteristics of RNNs**

1. **Sequential Data**: RNNs are designed to handle sequential data, which means they can process data that is
   ordered in time, such as words in a sentence, frames in a video, or timestamps in a time series.
2. **Recurrent Connections**: RNNs have recurrent connections, which allow the network to keep track of the state
   of the input sequence over time. This is achieved through the use of feedback loops in the neural network.
3. **Hidden State**: The hidden state of an RNN represents the internal state of the network at a particular time
   step. The hidden state is used to capture the sequential structure of the input data.

**How RNNs Process Data**

1. **Input Sequence**: An RNN receives an input sequence, which can be a sequence of words, frames, or timestamps.
2. **Forward Pass**: The input sequence is passed through the neural network in a forward pass, where each time
   step processes the current input and generates an output.
3. **Hidden State Update**: After processing each time step, the hidden state is updated based on the previous
   hidden state and the current input. This process is repeated for all time steps in the sequence.

**Types of RNNs**

1. **Simple RNN**: A simple RNN has a single layer of neurons and uses only the current input to update the hidden
   state.
2. **Long Short-Term Memory (LSTM) RNN**: An LSTM RNN is an improvement over the simple RNN, where it uses memory
   cells to handle long-term dependencies in sequential data.
3. **Gated Recurrent Unit (GRU) RNN**: A GRU RNN is similar to an LSTM RNN but with a different architecture that
   can handle longer sequences.

**Applications of RNNs**

1. **Natural Language Processing (NLP)**: RNNs are widely used in NLP applications such as language modeling,
   sentiment analysis, and machine translation.
2. **Speech Recognition**: RNNs are used in speech recognition systems to recognize spoken words and phrases.
3. **Time Series Prediction**: RNNs are used in time series prediction tasks, where the goal is to predict future
   values based on past data.

**Challenges of RNNs**

1. **Vanishing Gradients**: A common problem with RNNs is vanishing gradients, which can cause the network to lose
   information over time.
2. **Exploding Gradients**: Another challenge is exploding gradients, where the gradients become too large and can
   cause the network to diverge.

**Future Directions of RNNs**

1. **Deep Learning Architectures**: Researchers are exploring deeper learning architectures for RNNs, such as
   residual connections and attention mechanisms.
2. **Attention Mechanisms**: Attention mechanisms are being used in RNNs to selectively focus on certain parts of
   the input data.
3. **Transfer Learning**: Transfer learning is being explored for RNNs, where pre-trained models can be fine-tuned
   for specific tasks.

In summary, RNNs are a powerful tool for processing sequential data and have numerous applications in NLP, speech
recognition, and time series prediction. While they have several challenges to address, researchers continue to
explore new architectures and techniques to improve their performance and efficiency.

### **1. Basic RNN Structure**

```python
import torch
import torch.nn as nn


# Define a simple RNN model
class VanillaRNN(nn.Module):
    def __init__(self, input_size, hidden_size, output_size):
        super(VanillaRNN, self).__init__()
        self.rnn = nn.RNN(input_size, hidden_size, batch_first=True)
        self.fc = nn.Linear(hidden_size, output_size)

    def forward(self, x, hidden):
        # x: (batch_size, seq_len, input_size)
        out, hidden = self.rnn(x, hidden)
        # out: (batch_size, seq_len, hidden_size)
        out = self.fc(out)
        return out, hidden

    def init_hidden(self, batch_size):
        # Initialize hidden state
        return torch.zeros(1, batch_size, self.hidden_size)  # (num_layers, batch_size, hidden_size)
```

# RNN Implementation with NumPy

I'll implement a simple Recurrent Neural Network (RNN) from scratch using NumPy. This implementation will include
the basic RNN cell, forward and backward passes, and a simple text generation example.

```python
import numpy as np


class RNN:
    def __init__(self, input_size, hidden_size, output_size):
        """
        Initialize RNN parameters
        
        Args:
            input_size: Dimension of input features
            hidden_size: Dimension of hidden state
            output_size: Dimension of output
        """
        # Weights and biases
        self.Wx = np.random.randn(input_size, hidden_size) * 0.1  # Input to hidden
        self.Wh = np.random.randn(hidden_size, hidden_size) * 0.1  # Hidden to hidden
        self.Wy = np.random.randn(hidden_size, output_size) * 0.1  # Hidden to output

        self.bh = np.zeros((hidden_size,))  # Hidden bias
        self.by = np.zeros((output_size,))  # Output bias

        # For prediction
        self.losses = []
        self.predictions = []

    def forward(self, x, initial_hidden=None):
        """
        Forward pass through the RNN
        
        Args:
            x: Input sequence (seq_length, batch_size, input_size)
            initial_hidden: Initial hidden state (batch_size, hidden_size)
            
        Returns:
            outputs: Output sequence (seq_length, batch_size, output_size)
            hidden_states: Hidden states (seq_length, batch_size, hidden_size)
        """
        if initial_hidden is None:
            hidden = np.zeros((x.shape[1], self.hidden_size))
        else:
            hidden = initial_hidden

        outputs = []
        hidden_states = []

        for i in range(x.shape[0]):  # Iterate over sequence length
            # Forward through the RNN cell
            hidden = np.tanh(np.dot(x[i], self.Wx) + np.dot(hidden, self.Wh) + self.bh)
            output = np.dot(hidden, self.Wy) + self.by
            softmax_scores = self.softmax(output)

            outputs.append(softmax_scores)
            hidden_states.append(hidden)

        return np.array(outputs), np.array(hidden_states)

    def softmax(self, x):
        """Softmax activation function"""
        exp_x = np.exp(x - np.max(x, axis=1, keepdims=True))
        return exp_x / exp_x.sum(axis=1, keepdims=True)

    def backward(self, x, y, learning_rate=0.1):
        """
        Backward pass (gradient calculation and update)
        
        Args:
            x: Input sequence (seq_length, batch_size, input_size)
            y: Target sequence (seq_length, batch_size, output_size)
            learning_rate: Learning rate for gradient descent
        """
        # Initialize gradients
        dWx = np.zeros((self.input_size, self.hidden_size))
        dWh = np.zeros((self.hidden_size, self.hidden_size))
        dWy = np.zeros((self.hidden_size, self.output_size))
        dbh = np.zeros(self.hidden_size)
        dby = np.zeros(self.output_size)

        # We'll compute gradients in reverse order (backwards)
        # Start from the last time step
        m = x.shape[1]  # Batch size

        # Backward through the last time step
        for i in reversed(range(x.shape[0])):
            # Forward pass to get current outputs
            if i == 0:
                hidden = np.zeros((m, self.hidden_size))
            else:
                hidden = self.hidden_states[i - 1]  # Get hidden state from previous time step

            output = np.dot(hidden, self.Wy) + self.by
            softmax_scores = self.softmax(output)

            # Calculate output layer gradient
            dsoftmax = softmax_scores.copy()
            dsoftmax[range(m), y[i]] -= 1
            dsoftmax /= m

            # Output to hidden gradient
            dWy = np.dot(hidden.T, dsoftmax)
            dby = np.sum(dsoftmax, axis=0)

            # Hidden to output gradient
            dhidden_prev = np.dot(dsoftmax, self.Wy.T)

            # Now calculate hidden state gradient
            dhidden = dhidden_prev

            # Calculate input and hidden to hidden gradients
            dWx = np.dot(x[i].T, dhidden)
            dWh = np.dot(hidden.T, dhidden)
            dbh = np.sum(dhidden, axis=0)

            # Save gradients for this time step
            self.dWx = dWx
            self.dWh = dWh
            self.dWy = dWy
            self.dbh = dbh
            self.dby = dby

            # Update parameters
            self.Wx -= learning_rate * dWx
            self.Wh -= learning_rate * dWh
            self.Wy -= learning_rate * dWy
            self.bh -= learning_rate * dbh
            self.by -= learning_rate * dby

            # Store losses and predictions for evaluation
            self.losses.append(-np.log(np.mean(np.sum(np.choose(y[i], softmax_scores), axis=1))))
            self.predictions.append(softmax_scores)

        return self.losses

    def generate_text(self, seed_text, vocab, max_length=50):
        """
        Generate text using the trained RNN
        
        Args:
            seed_text: Starting text for generation
            vocab: Vocabulary dictionary mapping characters to indices
            max_length: Maximum length of generated text
            
        Returns:
            Generated text
        """
        # Convert seed text to indices
        seed_indices = [vocab[char] for char in seed_text]

        # Create input sequence (last character only)
        x = np.zeros((1, len(seed_indices), 1))
        for t in range(len(seed_indices)):
            x[0, t, 0] = seed_indices[t]

        # Forward pass to get initial hidden state
        _, hidden_states = self.forward(x)

        # Generate new text
        generated_text = seed_text

        for i in range(max_length):
            # Create input for next character
            x_next = np.zeros((1, 1, 1))
            x_next[0, 0, 0] = seed_indices[-1] if i == 0 else generated_text[-1]

            # Forward pass for next character
            output, _ = self.forward(x_next, initial_hidden=hidden_states[-1] if i > 0 else hidden_states[0])

            # Sample next character
            prediction = output[0, 0]
            next_char = np.random.choice(list(vocab.keys()), p=prediction)

            # Append to generated text
            generated_text += next_char

            # Update seed for next prediction
            seed_indices.append(vocab[next_char])

        return generated_text


# Example usage
if __name__ == "__main__":
```

# Hidden State in RNNs

The **hidden state** is one of the most fundamental concepts in Recurrent Neural Networks (RNNs). It acts as a
memory mechanism that allows the network to capture information from previous time steps and use it to influence
predictions at the current time step.

## What is a Hidden State?

In an RNN, the hidden state represents a compressed representation of the information processed so far in the
sequence. It's like a "memory" that carries forward contextual information from one time step to the next.

### Key Characteristics:

1. **Context Awareness**: The hidden state maintains information about the sequence history
2. **Continuity**: It provides a continuous link between time steps
3. **Memory**: It stores relevant information from past inputs

## How Hidden States Work

At each time step `t`, an RNN processes:

- **Input** `x_t`
- **Previous hidden state** `h_{t-1}`
- **Current hidden state** `h_t`
- **Output** `y_t`

The basic RNN equations are:

```
h_t = activation_function(Wx * x_t + Wh * h_{t-1} + bh)
y_t = Wo * h_t + by
```

Where:

- `Wx`, `Wh`, `Wo` are weight matrices
- `bh`, `by` are bias vectors
- `activation_function` is typically tanh or ReLU

## Types of Hidden States

1. **Vanilla RNN**: Single hidden state per time step (the most basic form)
2. **LSTMs**: Three hidden states (cell state, input gate, forget gate, output gate)
3. **GRUs**: Two hidden states (hidden state and memory cell)

## Why Hidden States Are Important

1. **Sequence Modeling**: They allow the network to process sequential data
2. **Temporal Dependencies**: They help capture long-range dependencies
3. **Contextual Understanding**: They provide context for current predictions

```python
        self.Wh -= learning_rate * d_Wh

        self.bh -= learning_rate * d_bh

        # Return gradient for previous hidden state
        return d_h_prev
```

# Backpropagation: The Heart of Neural Network Training

Backpropagation (short for "backward propagation of errors") is the fundamental algorithm used to train artificial
neural networks. It calculates the gradient of the loss function with respect to each weight in the network,
enabling efficient weight updates to minimize prediction errors.

---

## Why Backpropagation is Needed

Without backpropagation, neural networks couldn't learn from data. Here's the core problem it solves:

1. **The Credit Assignment Problem**:
    - Which weights contributed most to the error in a prediction?
    - How much should each weight be adjusted?

Backpropagation solves this by propagating error gradients backward through the network.

---

## How Backpropagation Works

### 1. Forward Pass

- Input data flows through the network
- Weights are multiplied by inputs
- Activations are computed layer by layer
- Output is compared to the target (loss is calculated)

### 2. Backward Pass (The Backpropagation Step)

- Compute gradients of the loss function with respect to each weight
- Use the chain rule of calculus to distribute error backward
- Update weights using gradient descent

### 3. Weight Update

- Weights are adjusted in the opposite direction of the gradient
- Formula: `weight = weight - learning_rate * gradient`

---

## Mathematical Foundation

Let's break down the core idea:

### Notation

- `L`: Loss function (e.g., mean squared error)
- `w`: Weight parameter
- `z`, `a`, `y`: Network outputs at different layers
- `x`: Input

### The Chain Rule in Action

Suppose we have a simple network:

```
a = σ(z₁)   // activation function
z₁ = w₁·x + b₁
L = (a - y)²  // loss
```

To update `w₁`, we need ∂L/∂w₁. Using the chain rule:

```
∂L/∂w₁ = ∂L/∂a * ∂a/∂z₁ * ∂z₁/∂w₁
```

### Step-by-Step Calculation

1. ∂L/∂a = 2(a-y)  (derivative of squared error)
2. ∂a/∂z₁ = σ'(z₁) (derivative of activation function)
3. ∂z₁/∂w₁ = x (input to the weight)

Final gradient: `∂L/∂w₁ = 2(a-y) * σ'(z₁) * x`

---

## Key Components of Backpropagation

### 1. Gradients

- Measure how much the loss changes with small weight adjustments
- Larger gradients mean bigger updates (but can cause instability)

### 2. Activation Functions

- Must be differentiable for backpropagation to work
- Common choices: ReLU, sigmoid, tanh

### 3. Loss Functions

- Must be differentiable to calculate gradients
- Examples: Mean Squared Error (MSE), Cross-Entropy

### 4. Optimization Algorithms

- Stochastic Gradient Descent (SGD)
- Adam, RMSprop, etc. (improved versions)

---

## Why It Matters

Backpropagation enables:

1. Training deep neural networks (hence "Deep Learning")
2. Automatic calculation of weight importance
3. Efficient updates using gradient information
4. The foundation of all modern neural network training

---

## Common Pitfalls

1. **Vanishing Gradients**: Deep networks may have very small gradients in early layers
    - Solution: Use ReLU activations, skip connections (ResNet)

2. **Exploding Gradients**: Gradients can become too large
    - Solution: Gradient clipping, weight regularization

3. **Non-Differentiable Parts**: Dropout layers are temporarily removed during training

---

## Implementation Tip

In practice, you don't implement backpropagation manually:

- Deep learning frameworks (TensorFlow, PyTorch) automatically compute gradients
- The chain rule is implemented via automatic differentiation

---

### **Backpropagation vs. Gradient Descent: What They Are**

1. **Gradient Descent**:
    - An **optimization algorithm**.
    - Updates parameters (weights/biases) iteratively to minimize a loss function.
    - Formula: `θ = θ - α * ∇J(θ)`, where:
        - `θ`: Parameters (e.g., `w` or `b`).
        - `α`: Learning rate.
        - `∇J(θ)`: Gradient of the loss function `J(θ)` with respect to `θ`.

2. **Backpropagation**:
    - A **method to compute gradients** efficiently.
    - Uses the chain rule of calculus to distribute the error backward through the network.
    - Works for any differentiable function, including deep neural networks.

---

### **Same in Linear Regression?**

Yes, they are **fundamentally the same process** in linear regression, but let’s clarify:

#### **Linear Regression Example**

Suppose we have a simple linear model:
\[
h_\theta(x) = \theta_0 + \theta_1 x
\]
and a loss function (e.g., Mean Squared Error):
\[
J(\theta_0, \theta_1) = \frac{1}{2m} \sum_{i=1}^m (h_\theta(x_i) - y_i)^2
\]

To train this model, we use **gradient descent**:

1. Compute the gradient of `J` with respect to `θ_0` and `θ_1`.
2. Update the parameters iteratively.

The gradient calculation for linear regression is straightforward (no need for backpropagation):
\[
\frac{\partial J}{\partial \theta_0} = \frac{1}{m} \sum_{i=1}^m (h_\theta(x_i) - y_i)
\]
\[
\frac{\partial J}{\partial \theta_1} = \frac{1 * \theta_1}{m} \sum_{i=1}^m (h_\theta(x_i) - y_i) \cdot x_i
\]

---

### **Same in Neural Networks?**

In **neural networks**, the process is **similar but more complex**:

1. **Gradient Descent** still updates parameters (weights/biases) to minimize the loss.
2. **Backpropagation** computes the gradients efficiently using the chain rule.

#### **Analogy**

- **Gradient Descent** is the "engine" that drives learning.
- **Backpropagation** is the "fuel" that provides the gradients.

# Stochastic Gradient Descent (SGD) in Logistic Regression

## What is Stochastic Gradient Descent (SGD)?

Stochastic Gradient Descent is an optimization algorithm that updates model parameters using a single example (or
a small batch) at a time, rather than all training examples like Batch Gradient Descent.

## SGD in Logistic Regression

Let's implement a logistic regression model using SGD:

```python
import numpy as np
import matplotlib.pyplot as plt

# Generate synthetic data
np.random.seed(42)
X = np.concatenate([np.random.normal(-4, 1, 100), np.random.normal(4, 1, 100)])
y = np.concatenate([np.zeros(100), np.ones(100)])
X = X.reshape(-1, 1)  # Reshape to 2D array

# Add bias term
X = np.hstack([np.ones(X.shape[0])[:, np.newaxis], X])

# Hyperparameters
learning_rate = 0.01
n_iterations = 100
batch_size = 1  # Stochastic gradient descent

# Initialize parameters
theta = np.random.randn(2, 1)

# Track loss
losses = []

# SGD implementation
for i in range(n_iterations):
    # Shuffle data each iteration
    idx = np.random.permutation(X.shape[0])
    X_shuffled = X[idx]
    y_shuffled = y[idx]

    for j in range(X_shuffled.shape[0]):
        # Single sample (stochastic)
        x_j = X_shuffled[j, :-1]  # Features (bias and feature)
        y_j = y_shuffled[j]  # Target

        # Forward pass
        z = np.dot(theta.T, x_j)
        y_pred = 1 / (1 + np.exp(-z))

        # Backward pass (compute gradient)
        gradient = (y_pred - y_j) * y_pred * (1 - y_pred) * x_j
        gradient = gradient.reshape(-1, 1)

        # Update parameters
        theta -= learning_rate * gradient

    # Compute loss for plotting
    losses.append(compute_loss(X, y, theta))

print("Final parameters:", theta)

# Plot loss
plt.plot(losses)
plt.title('Loss over Iterations (SGD)')
plt.xlabel('Iteration')
plt.ylabel('Loss')
plt.show()


# Function to compute logistic loss
def compute_loss(X, y, theta):
    m = X.shape[0]
    z = np.dot(X, theta)
    y_pred = 1 / (1 + np.exp(-z))
    loss = (-y * np.log(y_pred) - (1 - y) * np.log(1 - y_pred)).sum() / m
    return loss
```

## Key Differences from Batch Gradient Descent

1. **Data Processing**:
    - Batch: Processes all training examples for each parameter update
    - SGD: Processes one example at a time for each parameter update

2. **Computational Efficiency**:
    - Batch: Requires storing entire dataset in memory
    - SGD: Can be more memory-efficient for large datasets

3. **Convergence**:
    - Batch: Converges to minimum
    - SGD: Converges to a neighborhood of the minimum (noisy updates)

## Advantages of SGD

1. **Faster convergence** in many cases due to frequent updates
2. **Noise helps escape local minima**
3. **Better for online learning** (streaming data)
4. **Natural regularization** (averaging over iterations)

## Disadvantages of SGD

1. **Noisy updates** can make convergence less stable
2. **Requires careful tuning** of learning rate
3. **May require more iterations** to reach the same level of accuracy

## Common Variants

1. **Mini-batch SGD**: Uses small batches (e.g., 32, 64, 128 samples)
2. **Momentum SGD**: Adds velocity to parameter updates
3. **RMSProp**: Adjusts learning rate per parameter
4. **Adam**: Combines momentum and RMSProp

## When to Use SGD

- Real-time learning scenarios
- When you want to escape local minima
- When computational resources are limited

# Batch Normalization vs. Layer Normalization in Transformers

While Batch Normalization (BatchNorm) is a fundamental technique in deep learning, Layer Normalization (LayerNorm)
is particularly important in transformer architectures. Let's explore how LayerNorm is implemented and used in

```python
import torch.nn.functional as F


class TransformerLayer(nn.Module):
    def __init__(self, d_model, num_heads, dim_feedforward):
        super().__init__()
        self.self_attn = nn.MultiheadAttention(d_model, num_heads)
        self.linear1 = nn.Linear(d_model, dim_feedforward)
        self.linear2 = nn.Linear(dim_feedforward, d_model)

        # Layer Normalization layers
        self.norm1 = nn.LayerNorm(d_model)
        self.norm2 = nn.LayerNorm(d_model)

        # Feedforward layer with dropout
        self.activation = nn.ReLU()
        self.dropout1 = nn.Dropout(0.1)
        self.dropout2 = nn.Dropout(0.1)

    def forward(self, x, mask=None):
        # Self-attention with LayerNorm
        attn_output = self.self_attn(x, x, x, attn_mask=mask)[0]
        x = self.norm1(x + attn_output)  # Pre-LayerNorm architecture
        x = self.activation(self.linear1(x))
        x = self.norm2(x + self.dropout2(self.linear2(x)))
        return x
```

# Vectorization in Linear Regression and Gradient Descent

Vectorization is a powerful technique that replaces loops with element-wise or matrix operations, leading to more
efficient and concise code. Let's implement linear regression and stochastic gradient descent (SGD) using

```python
import numpy as np
import matplotlib.pyplot as plt


class VectorizedLinearRegression:
    def __init__(self, learning_rate=0.01, n_iterations=100):
        self.learning_rate = learning_rate
        self.n_iterations = n_iterations
        self.theta = None
        self.losses = []

    def fit(self, X, y):
        # Add bias term (column of ones)
        X_b = np.c_[np.ones((X.shape[0], 1)), X]

        # Initialize parameters
        self.theta = np.random.randn(X_b.shape[1], 1)

        # SGD implementation with vectorization
        for i in range(self.n_iterations):
            # Vectorized forward pass
            y_pred = X_b @ self.theta

            # Vectorized loss calculation (Mean Squared Error)
            loss = np.dot(y_pred - y, y_pred - y) / (2 * len(y))
            self.losses.append(loss)

            # Vectorized gradient calculation
            gradient = np.dot(X_b.T, y_pred - y) / len(y)

            # Vectorized parameter update
            self.theta -= self.learning_rate * gradient

            # Print progress
            if i % 10 == 0:
                print(f"Iteration {i}: Loss = {loss:.4f}")

        return self

    def compute_loss(self, X, y, theta):
        m = len(y)
        y_pred = X @ theta
        return np.dot(y_pred - y, y_pred - y) / m
```

```python
        self.theta = np.random.randn(n, 1)
        self.losses = np.zeros(self.n_iterations)

            # Shuffle data
            idx = np.random.permutation(m)
            X_shuffled = X[idx]
            y_shuffled = y[idx]

            # Batch gradient from vectorized operations
            for i in range(0, m, self.batch_size):
                X_batch = X_shuffled[i:i + self.batch_size]
                y_batch = y_shuffled[i:i + self.batch_size]

                # Vectorized predictions
                y_pred = X_batch @ self.theta

                # Vectorized gradient calculation
                gradient = X_batch.T @ (y_pred - y_batch) / self.batch_size

                # Vectorized parameter update
                self.theta -= self.learning_rate * gradient

            # Calculate loss for monitoring
            y_pred_full = X @ self.theta
            self.losses[it] = np.dot(y_pred_full - y, y_pred_full - y) / m

        return self
```

   implementations.

## Best Practices for Vectorization in CNNs

   automatically.

2. **Avoid Python Loops**: Minimize explicit loops in your code; use vectorized operations instead.

3. **Batch First**: Design your models to process batches first, channels last (for computer vision tasks).

5. **Memory Layout**: Be mindful of memory layout (contiguous memory access is faster).

# Hyperparameters in Neural Networks

Hyperparameters in neural networks are configuration settings that are **set before training** and define the
structure of the model and the learning process. They are not learned during training. Here are the main
categories:

## 1. Architecture Hyperparameters

- **Number of layers**: Hidden layers, convolutional layers, recurrent layers
- **Neurons/units**: Number of units in each layer
- **Layer types**: Dense (fully connected), convolutional, recurrent, etc.
- **Activation functions**: ReLU, sigmoid, tanh, etc.
- **Kernel size**: For convolutional layers (e.g., 3x3, 5x5)
- **Dropout rate**: Fraction of neurons to randomly deactivate during training
- **Batch normalization**: Whether to use and its parameters

## 2. Training Hyperparameters

- **Learning rate**: How much to update weights at each iteration (e.g., 0.001)
- **Optimizer**: Adam, SGD, RMSprop, etc.
- **Loss function**: Cross-entropy, MSE, etc.
- **Epochs**: Number of full passes through the training data
- **Batch size**: Number of samples per gradient update
- **Number of iterations**: Total steps for one epoch
- **Momentum**: For SGD to accelerate convergence (e.g., 0.9)
- **Weight decay**: L2 regularization parameter
- **Learning rate schedule**: How learning rate changes over time

## 3. Regularization Hyperparameters

- **L1/L2 regularization**: Strength of weight decay
- **Dropout**: Fraction of inputs to randomly drop during training
- **Batch normalization**: Whether to use and its parameters

## 4. Other Hyperparameters

- **Initializer**: Weight initialization method (e.g., He, Xav, Glorot)
- **Early stopping**: Patience value for stopping training
- **Callback functions**: Model checkpoint saving, etc.
- **Data augmentation**: Transformations applied to training data
